### 🔑 깃허브 토큰(Token) 생성 및 로그인 방법

1. **깃허브 홈페이지 접속 및 설정으로 이동**
   * 웹 브라우저에서 [GitHub Developer Settings](https://github.com/settings/tokens) 페이지로 바로 접속합니다.
   * (혹은 깃허브 로그인 후 우측 상단 프로필 이미지 클릭 -> **`Settings`** -> 맨 하단 **`Developer settings`** -> **`Personal access tokens`** -> **`Tokens (classic)`** 클릭)

2. **토큰 발급받기**
   * 우측 상단 **`Generate new token`** -> **`Generate new token (classic)`**을 클릭합니다.
   * **Note** (이름): 아무 이름이나 입력합니다. (예: `mac-terminal`)
   * **Select scopes** (권한 설정): 맨 위의 **`repo`** 체크박스에 체크합니다. (코드를 올리고 내려받는 권한입니다.)
   * 페이지 맨 아래로 내려가 녹색 **`Generate token`** 버튼을 누릅니다.

3. **토큰 복사하기**
   * 화면에 임의의 긴 문자열(예: `ghp_xxxx...`)로 된 토큰이 생성됩니다. 이 값을 **복사(Copy)**해 둡니다. 
   * *(⚠️ 이 토큰 값은 창을 닫으면 다시 볼 수 없으니 메모장 등에 따로 보관해 두셔도 좋습니다.)*

4. **터미널에서 다시 로그인 시도**
   * 터미널 창으로 돌아가 다시 `git push`를 입력합니다.
   * **Username**: `jbc6265` 입력 후 엔터
   * **Password**: 방금 복사한 **토큰 값(`ghp_...`로 시작하는 복사한 텍스트)**을 붙여넣기(Command + V)한 후 엔터를 누릅니다. *(⚠️ 터미널에서는 보안상 비밀번호를 입력하거나 붙여넣을 때 글자가 화면에 보이지 않는 것이 정상이니, 안 보이더라도 붙여넣고 바로 엔터를 누르시면 됩니다!)*

### 🤖 Antigravity
**403 에러 (Permission denied)**가 발생했습니다. 이 오류는 입력하신 토큰에 해당 저장소를 수정할 수 있는 **쓰기 권한(Write Permission)**이 누락되었을 때 발생합니다. 

이 문제를 해결하기 위해 다음 설정을 확인하고 재시도해 주세요.

---

### 1. 토큰 발급 시 `repo` 권한 체크 확인 (Classic 토큰 기준)
토큰을 새로 생성하실 때, 권한 목록(Select scopes)의 맨 위에 있는 **`repo` 체크박스에 체크**를 하셨는지 확인해 주세요. 
* 이 체크박스가 선택되어 있지 않으면 로그인에는 성공해도 파일을 올리는(Write/Push) 권한이 없어서 403 거부 에러가 발생합니다.
* 깃허브 [Developer Settings](https://github.com/settings/tokens)로 다시 들어가셔서 **`repo`** 항목 전체에 체크를 하고 토큰을 새로 생성하여 다시 시도해 주세요.

---

### 2. Fine-grained Token (세분화된 토큰)을 사용하신 경우
혹시 'Classic' 대신 최신 버전인 'Fine-grained Token'으로 생성하셨다면 아래 두 가지가 설정되었는지 확인해야 합니다:
1. **Repository access**: `All repositories` (모든 저장소) 혹은 `Only select repositories`에서 `household-ledger` 저장소를 지정했는지 확인.
2. **Permissions (권한)** -> **Repository permissions** -> **`Contents`** 권한을 **`Read and write` (읽기 및 쓰기)**로 설정했는지 확인. (기본값인 Read-only일 경우 403 에러가 납니다.)

## 📂 가계부 `index.html` 파일의 전체 구조

이 파일은 HTML, CSS, JavaScript가 한데 모여 있는 **단일 웹 페이지(Single Page App)** 형태로, 구조는 크게 세 영역으로 구분됩니다:

### 1. 외부 라이브러리 연동 (1~16번 줄)
* **폰트**: 고딕체 기반의 세련된 `Inter`, `Outfit`, `Noto Sans KR` 구글 폰트를 불러옵니다.
* **차트**: 데이터 분석 그래프 작성을 위해 **`Chart.js`** 라이브러리를 사용합니다.
* **CSV 파싱**: 사용자 가계부 데이터(CSV 파일)를 브라우저에서 동적으로 읽어들이기 위해 **`PapaParse`** 라이브러리를 탑재하고 있습니다.

### 2. 스타일 정의 (17~984번 줄 / `<style>`)
* **다크 글래스모피즘(Glassmorphism)**: 어두운 배경(`--bg-primary: #0b0f19`)과 반투명 카드 테마, 그리고 네온 보라/인디고 컬러 그라데이션으로 디자인되어 있습니다.
* **KPI 카드 스타일**: 수입(초록색), 지출(빨간색), 순수익(파란색), 저축률(주황색) 등을 하단 바 컬러 포인트로 강조합니다.
* **반응형 대기**: 모바일, 태블릿, PC 화면 크기에 맞춰 레이아웃이 유연하게 조절되는 그리드(Grid)와 플렉스(Flex) 속성을 사용합니다.

### 3. 화면 구조 및 스크립트 (985~2002번 줄 / `<body>` 및 `<script>`)
* **UI 레이아웃**:
  * **헤더**: 가계부 타이틀과 데이터 불러오기(CSV 업로드) 버튼.
  * **네비게이션 탭**: 대시보드 / 월별 분석 / 카테고리별 지출 분석 / 상세 내역 등 화면 전환 탭.
  * **콘텐츠 영역**: 지출 트렌드 차트(Chart.js), 카테고리 점유율 파이 차트, 최근 내역 표(Table) 등이 탭에 맞춰 배치되어 있습니다.
* **JavaScript 핵심 로직**:
  * CSV 데이터가 업로드되면 즉시 수입/지출을 분류하여 합산하고, 차트를 실시간으로 다시 그리는 이벤트 엔진이 내장되어 있습니다.

---

### 방법 1: 저장소 메인 화면의 `Deployments` 섹션 확인 (가장 직관적)
1. 브라우저에서 사용자님의 GitHub 저장소 메인 화면(`https://github.com/jbc6265/household-ledger`)으로 들어갑니다.
2. 화면 **우측 사이드바**를 아래로 조금 내려보면 **`Deployments`**라는 섹션이 있습니다.
3. 거기에서 **`github-pages`** 문구를 클릭합니다.
4. 지금까지 **언제, 어떤 커밋(버전)으로 배포가 되었는지 전체 이력**이 타임라인 형태로 나타납니다. 
   * 각 이력마다 있는 `View deployment` 버튼을 누르면 해당 시점의 사이트로 이동할 수 있습니다.

---

### 방법 2: `Actions` 탭 확인 (배포 과정 및 상태 모니터링)
1. 저장소 상단 메뉴 중 **`Actions`** (재생 버튼 모양 아이콘) 탭을 클릭합니다.
2. 왼쪽 목록에서 **`pages-build-deployment`** 워크플로우를 클릭하거나 전체 목록을 봅니다.
3. `git push`를 할 때마다 깃허브가 서버를 빌드한 작업 목록이 쭉 나옵니다:
   * **초록색 체크 (✔️)**: 배포 성공
   * **황색 회전 아이콘 (⏳)**: 현재 배포 진행 중
   * **빨간색 X 표시 (❌)**: 배포 실패 (오류 로그 확인 가능)
4. 실패했을 경우 해당 항목을 클릭하면 어떤 코드 때문에 에러가 났는지 상세 기록을 볼 수 있습니다.

---

### 방법 3: `Commits` 이력 확인 (실제 코드 변경점 추적)
배포된 웹사이트의 각 버전은 결국 코드 커밋(Commit)과 1:1로 일치합니다.
1. 저장소 메인 화면의 파일 목록 바로 위에 있는 **`commits`** 링크(예: `15 commits`)를 클릭합니다.
2. 지금까지 수정하고 남긴 커밋 메시지와 버전 히스토리가 순서대로 나타납니다.
3. 특정 커밋을 클릭하면 **이전 버전과 비교하여 정확히 어떤 코드 라인이 추가되거나 삭제되었는지(Diff)** 한눈에 볼 수 있습니다.

이 기능들을 활용하시면 배포 도중 발생한 에러를 추적하거나 과거의 특정 상태로 웹사이트를 롤백(되돌리기)할 때 매우 편리합니다!

### 1. 웹사이트 주소(URL) 구성의 차이

배포하는 페이지는 크게 두 가지 형태로 나뉩니다:

* **사용자 대표 페이지 (대표 도메인 - 딱 1개만 가능)**
  * 주소 형식: `https://jbc6265.github.io/`
  * 생성 방법: 저장소 이름을 내 깃허브 아이디를 포함해 **`jbc6265.github.io`**라고 똑같이 만들면, 이 주소로 대표 웹사이트가 배포됩니다.
* **프로젝트 페이지 (무제한 가능)**
  * 주소 형식: `https://jbc6265.github.io/<저장소이름>/`
  * 예시: `https://jbc6265.github.io/household-ledger/` (가계부)
  * 예시: `https://jbc6265.github.io/press_pop/` (프레스 대시보드)
  * 생성 방법: 저장소 이름을 자유롭게 짓고 Pages를 활성화하면 뒤에 슬래시(`/`)와 함께 개별 사이트로 무제한 배포됩니다.
